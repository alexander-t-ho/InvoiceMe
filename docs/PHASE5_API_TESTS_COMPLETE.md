## Test Scenarios Covered

### Success Scenarios ✅
- Creating entities (Customer, Invoice, Payment)
- Updating entities
- Deleting entities
- Retrieving entities by ID
- Listing entities with pagination
- Filtering lists (by status, customer)
- Business operations (add line items, mark as sent, record payments)
- Status transitions (DRAFT → SENT → PAID)

### Error Scenarios ✅
- **Validation Errors (400):**
  - Invalid email format
  - Missing required fields
  - Invalid date constraints
  - Invalid numeric values (negative, zero)
  - Constraint violations

- **Business Rule Violations (400):**
  - Cannot update SENT invoices
  - Cannot send invoice without line items
  - Cannot record payment for DRAFT invoice
  - Payment exceeds balance
  - Cannot delete customer with invoices

- **Not Found Errors (400):**
  - Non-existent customer
  - Non-existent invoice
  - Non-existent payment

### Pagination Tests ✅
- Page-based pagination
- Page size configuration
- Total elements and pages calculation
- Has next/previous indicators
- Multiple pages

## Test Implementation Details

### Test Framework
- **Spring Boot Test** - Full application context
- **MockMvc** - HTTP request/response testing
- **Jackson ObjectMapper** - JSON serialization/deserialization
- **H2 In-Memory Database** - Isolated test database
- **Transactional** - Tests rollback after execution

### Test Structure
Each test class follows this pattern:
1. **Setup** - Creates test data in `@BeforeEach`
2. **Test Methods** - Individual test scenarios
3. **Assertions** - Validates HTTP status codes and response bodies

### HTTP Status Code Validation
- ✅ 201 Created - For POST requests creating resources
- ✅ 200 OK - For successful GET/PUT requests
- ✅ 204 No Content - For successful DELETE requests
- ✅ 400 Bad Request - For validation and business rule errors
- ✅ 404 Not Found - For non-existent resources (mapped to 400 in our implementation)

### Response Body Validation
- ✅ JSON structure validation
- ✅ Field value validation
- ✅ Array length validation
- ✅ Nested object validation
- ✅ Error message validation
